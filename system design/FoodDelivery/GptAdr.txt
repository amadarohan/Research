ADR-001: مدیریت سفارش با Saga Pattern

Title: ADR-001: استفاده از Saga Pattern برای مدیریت چرخه سفارش

Status: Proposed

Date: 2026-06-15

Context

ثبت سفارش شامل چندین سرویس مستقل است:

Order Service
Restaurant Service
Payment Service
Courier Service

استفاده از Distributed Transaction باعث:

Coupling شدید
Latency بالا
کاهش Availability

می‌شود.

Decision

فرآیند سفارش بر اساس Saga Pattern پیاده‌سازی شود.

مراحل:

Create Order
Reserve Restaurant Capacity
Process Payment
Assign Courier
Confirm Order

در صورت شکست هر مرحله، عملیات جبرانی اجرا شود.

Consequences
Positive
Scalability بالا
Fault Isolation
مناسب برای Microservices
Negative / Trade-offs
Eventual Consistency
مدیریت پیچیده‌تر خطاها
نیاز به Monitoring پیشرفته
Alternatives Considered
Alternative 1: Distributed Transaction (2PC)

رد شد زیرا مقیاس‌پذیر نیست.

Alternative 2: Synchronous Chaining

رد شد زیرا Availability را کاهش می‌دهد.

Compliance
Saga Monitoring
Compensation Validation

ADR-003: Real-Time Location Tracking Architecture

Status: Proposed

Date: 2026-06-15

Context

کاربران باید موقعیت پیک را به صورت لحظه‌ای مشاهده کنند.

ارسال اطلاعات مکان از طریق Polling سنتی:

مصرف پهنای باند زیاد
Latency بالا
تجربه کاربری ضعیف

ایجاد می‌کند.

Decision

سیستم از Event Streaming استفاده کند.

فناوری‌های پیشنهادی:

Server Sent Events
Kafka / Pulsar

Consequences
Positive
Latency پایین
تجربه کاربری بهتر
مقیاس‌پذیری بالا
Negative / Trade-offs
مدیریت Connectionها دشوارتر
هزینه زیرساخت بیشتر
Alternatives Considered
Alternative 1: Polling

رد شد زیرا کارایی مناسبی ندارد.

Alternative 2: Database Polling

رد شد زیرا فشار زیادی بر دیتابیس وارد می‌کند.

Compliance
Location Accuracy Monitoring
WebSocket Health Checks
