ADR-001
معماری Hybrid Feed Generation

Title: ADR-001: استفاده از Hybrid Feed Generation (Fan-out on Write + Fan-out on Read)

Status: Proposed

Date: 2026-06-15

Context

بیش از ۸۰٪ ترافیک Instagram مربوط به مشاهده Feed است.

استفاده از Push کامل:

برای کاربران مشهور (Celebrity)
میلیون‌ها Follower

باعث ایجاد Write Storm می‌شود.

استفاده از Pull کامل نیز Latency بسیار بالایی در زمان مشاهده Feed ایجاد می‌کند.

Decision

سیستم از Hybrid Fan-out استفاده خواهد کرد:

کاربران عادی → Fan-out On Write
کاربران Celebrity → Fan-out On Read

آستانه اولیه:

کمتر از 10K Follower → Push
بیشتر از 10K Follower → Pull
Consequences
Positive
Feed سریع برای اکثر کاربران
جلوگیری از انفجار عملیات نوشتن
Scalability بسیار بهتر
هزینه زیرساخت کمتر
Negative / Trade-offs
پیچیدگی بیشتر Feed Service
نیاز به Merge چند منبع داده
Debugging سخت‌تر
Alternatives Considered

Alternative 1: Fan-out On Write Only

رد شد زیرا برای Celebrityها مقیاس‌پذیر نیست.

Alternative 2: Fan-out On Read Only

رد شد زیرا Latency Feed را افزایش می‌دهد.

Compliance
Load Testing
Feed SLA Monitoring

ADR-002
ذخیره‌سازی Social Graph

Title: ADR-002: استفاده از Graph-Oriented Social Relationship Store

Status: Proposed

Date: 2026-06-15

Context

Instagram میلیاردها رابطه Follow دارد.

کوئری‌های پرتکرار:

Who follows me?
Who do I follow?
Mutual Connections
Suggested Friends

در RDBMS سنتی Joinهای بسیار سنگین ایجاد می‌شود.

Decision

Social Graph در Storage اختصاصی گرافی نگهداری شود.

مانند:

TAO-like Graph Store
Neo4j-like Architecture

User Service مالک کامل Graph خواهد بود.

Consequences
Positive
Latency بسیار پایین
توسعه Recommendation آسان‌تر
Extensibility بهتر
Negative / Trade-offs
پیچیدگی بیشتر زیرساخت
نیاز به Replication تخصصی
Alternatives Considered

Alternative 1: Relational Database

رد شد به دلیل هزینه بالای Traversal.

Alternative 2: Shared User Table

رد شد زیرا Scalability مناسبی ندارد.

Compliance
Graph Query Benchmark
Cache Hit Rate Monitoring
