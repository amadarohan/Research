ADR-001: معماری Distributed Inverted Index

Status: Proposed

Date: 2026-06-15

Context

موتور جستجو باید میلیاردها سند را ایندکس کند.

ذخیره کل Index در یک Node باعث:

محدودیت حافظه
محدودیت IOPS
عدم تحمل خطا

خواهد شد.

Decision

ایندکس به صورت Sharded Distributed Inverted Index نگهداری شود.

هر Shard:

Primary Replica
Secondary Replica

خواهد داشت.

Query Router مسئول مسیریابی Query به Shardهای مربوطه است.

Consequences
Positive
Scalability افقی
Fault Tolerance
افزایش ظرفیت ایندکس
Negative / Trade-offs
افزایش Network Overhead
پیچیدگی Rebalancing
نیاز به Replica Synchronization
Alternatives Considered
Alternative 1: Single Large Index

رد شد زیرا مقیاس‌پذیر نیست.

Alternative 2: Shared Database Search

رد شد زیرا Latency بالایی ایجاد می‌کند.

Compliance
Shard Health Monitoring
Replication Verification

ADR-002: Event-Driven Indexing Pipeline

Status: Proposed

Date: 2026-06-15

Context

Crawlerها روزانه میلیون‌ها صفحه را جمع‌آوری می‌کنند.

ایندکس‌سازی Sync باعث Bottleneck می‌شود.

Decision

Pipeline ایندکس‌سازی مبتنی بر Event طراحی شود.

مراحل:

Crawl
Parse
Extract
Rank Signals
Index

از طریق Event Bus منتقل شوند.

Consequences
Positive
Extensibility بالا
پردازش مستقل مراحل
Fault Isolation
Negative / Trade-offs
Eventual Consistency
Monitoring پیچیده‌تر
Alternatives Considered
Alternative 1: Monolithic Indexer

رد شد زیرا Bottleneck ایجاد می‌کند.

Alternative 2: Scheduled Batch Indexing

رد شد زیرا Freshness نتایج کاهش می‌یابد.

Compliance
DLQ Monitoring
Event Versioning
